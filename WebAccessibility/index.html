Web Accessibility

Why Am I creating This Project?

I believe that the web should be for everyone. Now that I am creating tools for people to use, I want to make damn sure that as many people find joy in my projects as possible. This site is intended to be both an information repository from my learnings, as well as a project pen. My goal is to provide knowledge of what accessibility means, and how we as developers can accommodate those marginalized. These ideas are influenced by the Website Accessibility course taught by Jon Kuperman at frontendmasters.com. I hope you enjoy, and please reach out to me on twitter or LinkedIn if you would like to connect!

Disability Types

Designing for accessibility can make a world of different to those users who have disabilities. Disabilities may include: blindness, low vision, color blindess, deaf, cognitive disorders, and motor impairment. Developers have an opportunity to enable those with disabilities to actively participate on the web. We live in a time where the web is the only thing that can safely connect us to those we care for. Now is a time more than ever that we must advocate for those users. Here is a sample of how some people may experience the web

Keyboard Navigation

Many users will navigate your site with only the keyboard. It is critical that your web apps accommodate this style of navigation. A few things to consider while building your web app is to always enable interactive items to show focus. Focus is the primary visual que to display where a user is interacting with your web app. Another great feature is to include skip links. Skip links enable the user to bypass the content of the introduction of the site and get to the main content.


Below is a simple example of how a user might navigate a site with a keyboard. Use TAB and SHIFT + TAB to move right and left respectfully.

  1. Try to navigate to the start button using only the keyboard
  2. Navigate to the key buttons and spell your name using the virtual keyboard. Use TAB to move right and SHIFT+TAB to move left. Press enter to submit to the entry field.
  3. After you are done. Navigate to the stop button and press enter.
  4. Send me your best time @Councit, and let me know how you did!
